## M1 — Validate Warm Pool (diagnostic)

```bash
kubectl -n kube-system describe daemonset aws-node | grep -E "WARM_IP_TARGET|MINIMUM_IP_TARGET|WARM_ENI_TARGET"
    #   MINIMUM_IP_TARGET:                      0
    #   WARM_ENI_TARGET:                        0
    #   WARM_IP_TARGET:                         1
```

- The `VPC CNI` (`aws-node DaemonSet`) **pre-attaches** `ENIs` and **pre-allocates** `IPs` on each node so pods can start without waiting for an EC2 API call.
- Three env vars govern that "warm pool":

| Variable            | Meaning                                                     | Default |
| ------------------- | ----------------------------------------------------------- | ------- |
| `WARM_IP_TARGET`    | keeping a certain number of IPs available dynamically       | unset   |
| `MINIMUM_IP_TARGET` | minimum number of IPs is always available.                  | unset   |
| `WARM_ENI_TARGET`   | keeps a certain number of ENIs available for pod assignment | 1       |
